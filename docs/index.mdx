---
title: "clob-server"
description: "A self-hostable paper-trading platform built on top of the clob matching engine."
---

**clob-server** turns the [`clob`](https://github.com/thorlaidanegg/clob)
matching-engine library into a complete trading backend: REST + WebSocket APIs,
API-key auth, virtual-credit wallets, position & PnL tracking, a leaderboard, a
real-time market-data feed, Postgres persistence, Redis caching, and a Kafka
event bus — all in a **single binary** whose behaviour is selected by one
environment variable, `ROLE`.

```bash
ROLE=all docker compose up      # the whole stack in one process, no Kafka needed
```

<Note>
  **Scope:** virtual credits only — no real money, margin, or leverage. The
  design lets a real-money layer be added later by swapping two components (the
  pre-order hook and the settlement handler) without touching the engine or API.
</Note>

## What you get

<CardGroup cols={2}>
  <Card title="One binary, eight roles" icon="layers">
    `engine`, `gateway`, `settlement`, `portfolio`, `leaderboard`, `feed`,
    `booksnapshot`, or `all` — selected by the `ROLE` env var.
  </Card>
  <Card title="Real order types" icon="list-checks">
    Limit, market, stop, stop-limit, iceberg; TIF GTC / IOC / FOK / GTD / DAY.
  </Card>
  <Card title="Exact money math" icon="calculator">
    Fixed-point decimals everywhere — `BIGINT` in Postgres, decimal strings on
    the wire. Never `float64`.
  </Card>
  <Card title="Event-driven & idempotent" icon="repeat">
    Engine → bus → workers. Each worker is crash-safe via per-market offsets
    committed in the same transaction as its write.
  </Card>
</CardGroup>
